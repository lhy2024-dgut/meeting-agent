2026-05-26 01:50 | 任务 3.2（模板美化）详细实现说明
============================================================

【任务 3.2 目标】
  让用户能在导出 Word/PDF 时选择不同风格的模板，且 PDF 不再出现中文方框乱码。
  拆分为 4 个子任务：制作模板源文件 → PDF 中文字体注册 → 导出链支持选模板 → 前端预览。

============================================================
│ 子任务 1：制作模板源文件
│ 文件：scripts/setup_templates.py（新增）
│ 思路：用 python-docx 生成 Word 模板文件（.docx），里面用
│       {{title}} {{minutes_content}} 这类占位符标记动态内容；
│       用 PyMuPDF 生成 PDF 骨架文件，同样含占位符；
│       用 Pillow 生成预览缩略图，方便前端展示。
============================================================

  ▸ 为什么新增这个脚本？
    模板文件是二进制文件（.docx/.pdf），不能直接在代码里写。
    必须用一个脚本一次性生成它们，放到 storage/templates/source/ 下，
    后续 ExportChain 读的就是这些静态文件。

  ▸ 代码逻辑：

    [第49-66行] _add_cover(doc, title, subtitle)
      → 在 Word 文档开头插入 6 个空行 + 居中大标题 + 副标题（日期），
        然后分页。这就是模板的封面页。
      → 问：为什么不用 Word 的封面功能？
        答：因为 docxtpl 渲染时封面占位符可能被覆盖，手动构造更可控。

    [第69-84行] _add_heading_styled(doc, text, level)
      → 统一管理标题样式。level=1 用 18pt 深色粗体，
        level=2 用 14pt 深蓝粗体。保证两套模板标题风格一致。
      → 这样后续如果要改所有模板的标题样式，只需改这一个函数。

    [第112-125行] _create_base_doc(title, subtitle)
      → 所有模板的公共骨架：创建空白文档 → 加封面 → 加目录占位页。
        返回 Document 对象，子任务各自的模板函数往里面填内容块。
      → 用"模板方法模式"：公共部分写一次，差异部分由子函数补充。

    [第132-166行] create_default_docx() — 默认模板（商务稳重）
      → 调用 _create_base_doc() 得到骨架文档，
        然后依次添加三个分节（带分页符）：
          一、会议纪要 → {{minutes_content}}
          二、待办事项 → {{action_items}}
          三、会议决议 → {{resolutions}}

                  pdf_files[stem] = f

          for name in sorted(all_names):

问题现象：

    OllamaChatModel._generate() → 返回 ChatResult(ChatGeneration(message=AIMessage))

      raw = self.chain.invoke(params)

问题 4：LLM 返回 AIMessage 导致 minutes_chain.py 崩溃

【现象】

所以 self.chain.invoke(params) 返回 AIMessage 对象，不是 str。

然后 parse(AIMessage) 里 text or "" 评估为 AIMessage 对象（非空为 True），

文件：chains/minutes_chain.py

改动 1 — 第 70-71 行：
  原来：raw = self.chain.invoke(params)
  改为：
    raw = self.chain.invoke(params)
    raw_text = raw.content if hasattr(raw, 'content') else str(raw)

改动 2 — 第 77 行：
  原来：action, resolutions, minutes = self.parser.parse(raw)
  改为：action, resolutions, minutes = self.parser.parse(raw_text)

设计考虑：
- hasattr(raw, 'content') 是防御性检查
- raw_text 在 try 块内赋值，不影响 except OllamaLLMError
- 共改 3 行，不改变 parse() 逻辑

2026-06-09 | 再次修复 PDF 字体 — fontfile 不嵌入字体改用 fontname="china-s"
============================================================

问题现象:
  用户反馈"你自己看看你修复好的pdf是什么鬼"，
  之前的两轮修复（fontname冲突修复 + insert_textbox自动换行）都没解决根因。

根因分析:
  1. insert_textbox(fontfile=msyh.ttc) 虽然用系统字体做文字排布计算，
     但 PyMuPDF 不会把 fontfile 指定的字体嵌入 PDF 文件中。
  2. 查看生成 PDF 的原始结构 (/Resources/Font)，只引用了 Helvetica，
     没有中文字体。
  3. PDF 查看器只能用 Helvetica 渲染中文 → Helvetica 没有中文字形 →
     字形缺失 + 宽度计算错误 → 字叠在一起。

修复:
  engines/pdf_engine.py — 全部 insert_textbox() 调用改为
  fontname="china-s" (PyMuPDF 内置 Adobe Sans China 字体)，
  不再依赖系统字体文件 (msyh.ttc)。

  改前: insert_textbox(rect, text, fontfile=self._cjk_font_path, ...)
  改后: insert_textbox(rect, text, fontname="china-s", ...)

  "china-s" 是 PyMuPDF 内置的 CID 中文字体，渲染时自动嵌入 PDF，
  任何 PDF 查看器都能正确渲染中文。

  __init__ 中移除了 fitz.Font() 加载尝试（TTC 字体的 idx 参数在
  PyMuPDF 1.27.2 的 fitz.Font() 中不支持）。

影响范围:
  - 只改了 engines/pdf_engine.py
  - 不再依赖系统中文字体文件（但保留字体搜索列表以备后用）
  - 所有 insert_textbox() 调用统一改用 fontname="china-s"

2026-06-09 10:28 | 第四次修复 PDF — 根除字重叠/格式错乱（insert_textbox 返回值偏移 bug）
============================================================

问题现象:
  前面三次修复（fontname冲突、insert_textbox换行、china-s恢复）都没解决根因，
  PDF 仍然字叠字、格式混乱。text bbox 数据显示 y 坐标在页面上疯狂跳跃。

根因分析:
  PyMuPDF 1.27.2 的 page.insert_textbox() 返回值存在 bug：
    返回值 ≠ 文本底部 y 坐标（如文档所述）
    实际公式: 返回值 = rect_bottom - text_bottom - 4
  导致 _render_line() 中 result_y + 4 的计算完全错误，
  y 坐标在页面顶部（y≈50-80）和底部（y≈711-740）之间来回跳跃，
  产生严重字体重叠。

修复方案:
  完全抛弃 insert_textbox()，改用手动换行方案：
  1. fitz.Font("china-s") 创建字体对象 → font.text_length() 测量文本宽度
  2. 按字符逐个追加，超出 BODY_W 时换行 → 保证精确换行
  3. page.insert_text() 逐行渲染（返回值稳定可靠）
  4. 行距 LINE_SPACING=1.5（原来 result_y+4 ≈ 1.2 倍，太挤）
  5. 段间距 PARAGRAPH_GAP=6，不同段落之间有呼吸感

改动文件:
  engines/pdf_engine.py — 完整重写
    - _get_cjk_font() 改为缓存 fitz.Font 对象，供 text_length() 度量
    - 新增 _line_height() / _measure() / _wrap_text() 工具方法
    - _render_line() → _render_text()：手动换行 + insert_text 渲染
    - create_pdf_from_markdown() 全部重写
    - overlay_text_on_pdf() 复用 _render_text()
    - 删除了无用的 _render_line() 和 insert_textbox 相关代码

  - 长文本（50段）：6页，每页 y=32~738，无溢出 ✅

  - 超长单行（无空格断词）：逐字换行，7 span，无重叠 ✅
