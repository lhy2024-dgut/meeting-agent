# Bug 修复汇总

**更新日期：** 2026-07-14
**范围：** 账号登录、会议问答、Todo 闭环、统计页

## 已修复

- P0：JWT 使用可预测默认密钥、默认管理员密码可被猜测，存在账号接管风险。
- P0：会议问答会话未绑定创建者，其他已登录用户拿到会话 ID 后可读取问答结果。
- P0：退出登录或修改密码后，旧的 access Token 和 refresh Token 仍可继续使用。
- P1：旧格式会议纪要解析时会漏提取待办事项和决议，导致 Todo 自动同步不完整。
- P1：Todo 编辑后无法清空负责人和截止日期。
- P1：会议问答声明有超时兜底，但线程池会等待模型调用结束，实际请求仍会长期阻塞。
- P1：登录后的 `next` 参数未校验，存在跳转到外部地址的风险。
- P1：会议日期未保存用户填写的会议时间，历史列表和月度统计按处理时间统计。
- P1：会议环境固定保存为 `unknown`，统计页多人会议和环境分布长期失真。
- P1：前端 `npm run lint` 存在错误，根目录直接执行 `pytest` 会误收集运行时目录中的测试文件。
- P2：已取消 Todo 的复选框仍可点击，导致非法状态流转报错。
- P2：Todo 创建失败后表单被清空、编辑失败后退出编辑状态，用户输入可能丢失。
- P2：Todo 总览只加载前 50 场会议，较早会议的 Todo 不显示所属会议名称。
- P2：统计页环境饼图没有类别颜色和图例，无法直观看懂各分类。

## 本轮已修复

- P0：问答缓存按用户 ID 隔离，退出或鉴权失效时会清理全部问答缓存。
- P0：会议重生成只替换未人工修改的自动抽取待办，人工新增、编辑、状态和日志均保留。
- P1：失效 Token 可正常进入登录页；SSR 页面会重定向到登录页而非返回 `500`。
- P1：问答会话失效时清除本地会话并自动创建新会话。
- P1：Todo 审计操作者由服务端写入 `user:{id}`，客户端字段不再生效。
- P1：无说话人分离结果的会议标记为“未知”，环境分布和历史筛选会明确展示该类别。
- P2：Todo 状态变更后会刷新已展开的状态日志。
- P2：统计页新增 Todo 总数、完成率、逾期数和负责人分布。
- P2：E2E 默认使用独立测试账号，新增失效 Token 登录页回归用例；跨账号会话和重生成保留逻辑由后端回归测试覆盖。
